## 📂 Organização do Repositório

Para garantir a escalabilidade e a limpeza do projeto, este repositório é estruturado da seguinte forma:

1. **Modalidade:** Identificação se a atividade foi realizada em Dupla ou Trio;
2. **Nomenclatura Semântica:** Pastas separadas por atividade, tecnologia e período.
   * *Exemplo:* `ATV1_PHP_1Bim`
3. **Documentação de Entrega:** Cada pasta contém o código-fonte e um arquivo `README.md` específico com:
   * Objetivo da atividade;
   * Data de entrega;
   * Nomes dos integrantes e professores;
   * Instruções de execução e outras observações pertinentes.
